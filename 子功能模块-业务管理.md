# 放射影像算法研究部管理系统



## 子功能模块-业务管理



### 业务管理背景

部门主要负责放射影像功能和图像的技术研发（预研和产品化），其研发流程包括以下几个阶段：

1. 需求收集——预研准入TRL2，立项

   需求导入

   ——阶段1-需求收集的完成节点：需求评审（一般来说需评审通过后方可进行下一阶段，也有一些很明确的需求可直接进行功能构思阶段），需详细描述需求和来源，或上传关于需求的文档

2. 功能构思——TRL3，立项，可正式开始预研

   1. 临床调研
   2. 痛点分析
   3. 需求提炼
   4. 竞品分析
   5. 方案构思
   6. 可行性分析

   ——阶段2-功能构思的完成节点：功能构思评审（一般来说需评审通过后方可进行下一阶段，也有一些很明确的预研可直接进行方案预研阶段），需上传功能构思PPT、评审会议纪要等材料

3. 方案预研（部门核心工作）

   1. 技术调研
   2. 方案设计及评审
   3. 任务分解
   4. 子任务开发
   5. 仿真验证
   6. TRL6预研验收
   7. 文档撰写
   8. 文档评审

   ——阶段3-方案预研的完成节点：TRL6评审（含内部临床验证报告），严格要求评审通过后方可进行下一阶段的版本开发，需上传TRL6评审材料和评审会议纪要

4. 版本开发及验证

   1. 方案澄清
   2. 功能开发
   3. 功能测试
   4. 临床验证（含方案迭代优化）：关键输出：临床验证报告
   5. 知识传递：关键输出：知识传递，需上传知识传递材料和会议纪要
   6. TRL7验收（可产品上市）



其中阶段4是和产品开发相关的，也是上市发布前的关键，涉及到跨部门（临床和开发），因此可单独细化

1. 方案澄清

   1. 系统方案和算法文档撰写
   2. 系统方案和算法方案澄清

2. 功能开发

   1. 参数提交
   2. 联调
   3. 性能验证和优化

3. 功能测试

   BUG解决

   - 需求类BUG
   - 算法性能类BUG
   - 算法设计缺陷BUG
   - 其他类BUG

4. 临床验证

   - 临床反馈问题优化
   - 临床反馈需求落实

5. 产品上市

   上市后效果跟踪



### 要求

- 支持展示预研标准流程，以及每个环节的参考文档
- 对部门负责的每一个功能和图像技术点，都以该标准流程跟踪预研进展
